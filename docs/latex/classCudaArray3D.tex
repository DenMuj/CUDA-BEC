class Cuda\+Array3\+D$<$ T $>$}
3D CUDA device memory 

This class manages GPU memory for 3D arrays with automatic selection between pitched memory (optimized for 2D/3D access patterns) and linear memory (fallback). Memory layout follows row-\/major ordering\+: index = iz\texorpdfstring{$\ast$}{*}(ny\texorpdfstring{$\ast$}{*}nx) + iy\texorpdfstring{$\ast$}{*}nx + ix where nx is fastest-\/changing, nz is slowest-\/changing dimension.

The constructor will try to allocate pitched memory. If pitched allocation fails, it automatically falls back to linear memory. For 1D arrays (ny=nz=1), linear memory is used regardless of the flag.

This constructor attempts to allocate the most efficient memory layout\+:
\begin{DoxyItemize}
\item For true 3D arrays (nz \texorpdfstring{$>$}{>} 1, ny \texorpdfstring{$>$}{>} 1)\+: Uses cuda\+Malloc3D for pitched memory
\item For 2D arrays (nz = 1, ny \texorpdfstring{$>$}{>} 1)\+: Uses cuda\+Malloc\+Pitch for pitched memory ~\newline

\item For 1D arrays or when pitched fails\+: Falls back to linear cuda\+Malloc 
\end{DoxyItemize}
